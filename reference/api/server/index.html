    server — API Reference¶

    Python API for the MCP server package. The server is normally launched as a subprocess by the host, but all business logic is importable for testing and scripting.

    server.tools¶

    Tool implementations — pure functions called by the MCP server handlers in server/main.py. Each function maps directly to an MCP tool of the same name.

    server.tools ¶

    Tool implementations — called by the MCP server handlers in main.py.

    approve_plan(plan_id, plans_dir) ¶

    Transition a plan from pending → approved.

    archive_document(checksum, reason, registry_path, quarantine_dir, journal_path, archive_path) ¶

    Withdraw a document from active memory ("retirer de la mémoire").

    Flips the registry record's status to archived, moves its file to the quarantine dir (collision-safe, recorded in the undo journal so it stays reversible via undo_last), and appends an entry to the archive log. The document is never deleted. If the file is already gone, the status flip and log entry still happen (no move). Raises if the checksum is not recorded.

    build_graph(registry_path, events_path, graph_path) ¶

    Rebuild the knowledge graph from the registry + event journal and persist it.

    Pure projection: the result depends only on the current registry and events, so it can be regenerated at any time. Returns the graph as {nodes, edges}.

    compare_documents(path_a, path_b, max_chars) ¶

    Extract text from two files and return a unified diff between them.

    Reuses the markitdown/pypdf extraction path, so it works on PDF/Office as well as plain text. Each side is truncated to max_chars before diffing, so the diff reflects only the extracted (possibly truncated) text — handy for comparing successive versions (e.g. two COPIL slide decks). identical is True when the extracted texts match exactly.

    compress_quarantine(quarantine_dir, journal_path, delete_originals=True) ¶

    Losslessly bundle loose quarantined files into a verified zip archive.

    Collects every top-level regular file currently in quarantine_dir (skipping archives this tool itself produced), writes them into a single ZIP_DEFLATED archive together with a checksum manifest, and verifies the archive byte-for-byte against the originals. Only once verification passes are the originals optionally removed to reclaim space. The whole operation is journaled so undo_last can restore every file and delete the archive. Nothing is deleted before the archive is verified, and existing archives are never overwritten.

    Idempotent: re-running with no new loose files is a no-op.

    compute_checksum(path) ¶

    Compute the sha256 checksum of a file (chunk-streamed) as its unique id.

    create_dir(path) ¶

    Create a directory (and any parents); idempotent if it already exists.

    Collision-safe by construction: an existing directory is returned as-is rather than raising, so the op can be re-run. Raises if path already exists as a file (not a directory).

    create_event(sentence, date, events_path) ¶

    Append a verb-led, dated event to the project event journal.

    Distinct from the undo journal: this records project narrative (what happened and when), not reversible file operations. sentence should be a short, verb-led statement; date is the ISO YYYY-MM-DD the event occurred (null if unknown).

    create_file(path, content) ¶

    Write content to path; raises if the file already exists.

    create_plan(plans_dir) ¶

    Create a new empty plan and persist it to disk.

    execute_plan(plan_id, plans_dir, journal_path, registry_path=None) ¶

    Apply approved ops with per-op retry; hard-stop if >3 fail in one run.

    When registry_path points at a non-empty registry, each executed op also reconciles the matching document record's path (and status, for quarantine) so the checksum stays the identity while paths track moves. Registry-optional: a missing/empty registry is a silent no-op.

    extract_text(path, max_chars) ¶

    Extract plain text from a PDF or Office file via markitdown.

    find_duplicates(registry_path) ¶

    Return fuzzy candidate-duplicate clusters for the host to judge.

    find_modified_documents(registry_path) ¶

    Return groups sharing a title but differing in content (modified docs).

    get_actors(graph_path, salient_cap) ¶

    Return the project's main actors — top entities ranked from the persisted graph, capped at salient_cap. Build the graph first (build_graph).

    get_document(checksum, registry_path) ¶

    Return a single document record by checksum, or None if absent.

    get_graph(graph_path) ¶

    Return the persisted knowledge graph as {nodes, edges}; empty if never built.

    get_plan(plan_id, plans_dir) ¶

    Load and return a plan by plan_id.

    get_registry(registry_path) ¶

    Return the entire registry as a dict for the host to reason over.

    list_archived(archive_path) ¶

    Return all archived-document log entries in chronological order.

    list_dir(path) ¶

    Enumerate directory entries with size, type, and mtime.

    list_documents(registry_path) ¶

    Return all document records, oldest first.

    list_events(events_path) ¶

    Return all recorded project events in chronological order.

    list_plans(plans_dir) ¶

    Return all plans sorted by creation time.

    move_file(path, dest_dir) ¶

    Move a file to dest_dir, raising if the destination already exists.

    propose_move(path, dest_dir, plan_id, plans_dir) ¶

    Append a move op to an existing pending plan; eager collision check.

    propose_quarantine(path, plan_id, plans_dir, quarantine_dir) ¶

    Append a quarantine op to an existing pending plan; picks collision-safe dest.

    propose_rename(path, new_name, plan_id, plans_dir) ¶

    Append a rename op to an existing pending plan; eager collision check.

    read_file(path, max_chars) ¶

    Return file text up to max_chars characters.

    record_document(checksum, path, title, type, summary, provenance, date, entities, attributes, status, registry_path, profile) ¶

    Upsert an analyzed document into the registry, validated against the profile.

    type must be one of the active profile's document types. Entity role values, when present, must belong to the profile's role taxonomy. The author guardrail (only include people explicitly named, never inferred) is a prompt instruction, not enforced here.

    rename_file(path, new_name) ¶

    Rename a file in place, raising if the new name already exists.

    review_plan(plan_id, plans_dir) ¶

    Read-only deduplication and pre-flight check; never modifies the plan.

    undo_last(journal_path, plans_dir) ¶

    Revert the most recent journaled op; only removes the entry on success.

    update_file(path, content) ¶

    Overwrite or create content at path.

    write_folder_readme(folder, content) ¶

    Write LLM-composed prose to README.md inside a folder of the arborescence.

    Thin sink, like write_summary: the host composes a short description of what the folder holds and its role in the organized tree; this just persists it. Overwrites any existing README and creates the folder if needed.

    write_index(target_dir, journal_path) ¶

    Walk target_dir, emit INDEX.md (tree + journal changelog) and manifest.json.

    write_summary(target_dir, content) ¶

    Write LLM-composed prose to SUMMARY.md inside target_dir.

    server.registry¶

    Document registry — the engine's persistent, content-addressed memory.

    server.registry ¶

    Document registry — the engine's persistent, content-addressed memory.

    A single JSON store at registry_path (default .organizer/registry.json) holding one record per unique document, keyed by its sha256 checksum. Because the checksum is the identity, the path may change as files are organized while the record (and its accumulated analysis) survives.

    This module is deliberately profile-agnostic: validation of a document's type against the active profile lives in the tools layer, not here.

    DocumentRecord(checksum, path, title, type, summary, provenance, date=None, entities=list(), attributes=dict(), status='active', first_seen='', last_analyzed='') dataclass ¶

    One analyzed document. Identity = checksum; path tracks location.

    Registry(documents=dict()) dataclass ¶

    In-memory view of the document store, keyed by checksum.

    find_duplicates(threshold=0.6) ¶

    Fuzzy candidate duplicate clusters for the host to judge.

    Pre-grouping heuristic (server stays free of semantic judgement): cluster records by title-token Jaccard similarity within the same document type, or on an exact normalized-title match across types. Returns clusters of size > 1; the host decides "même information / susceptible d'être remplacé".

    find_modified() ¶

    Groups sharing a normalized title — same title, different content.

    Records are keyed by checksum, so any same-title group inherently has differing checksums: the "modifié récemment (même titre, checksum ≠)" case.

    update_path(old_path, new_path, status=None) ¶

    Reconcile a record's location after a move/quarantine (G5).

    Matches the record whose current path equals old_path and rewrites it to new_path (optionally flipping status). Returns None if no record currently lives at old_path (registry-optional reconcile).

    upsert(record) ¶

    Insert or update by checksum; preserve first_seen, refresh last_analyzed.

    load(registry_path) ¶

    Load the registry; returns an empty Registry if the file does not exist.

    save(registry, registry_path) ¶

    Persist the registry to disk (pretty JSON, Unicode preserved).

    server.profile¶

    Domain profile loader — reads a .toml file and exposes typed accessors.

    server.profile ¶

    Domain profile: declarative config that adapts the engine to a document corpus.

    A profile externalizes everything domain-specific (document-type vocabulary, entity/role model, extraction guardrails, naming, synthesis template, output sinks) so the same engine can serve different kinds of document piles by swapping a TOML file. The bundled is_it_project profile is profile #1.

    DocumentType(id, label, description='') dataclass ¶

    One entry of a profile's controlled document-type vocabulary.

    Profile(name, description, document_types, entity_kinds, role_taxonomy, salient_cap, extraction_required, extraction_optional, synthesis_template, synthesis_title, synthesis_sections, synthesis_instructions, naming_convention, naming_instructions, sinks_default) dataclass ¶

    A loaded, validated domain profile with typed accessors.

    document_type_ids() ¶

    Valid type values for this profile (used to validate records).

    entity_roles() ¶

    Allowed entity roles (e.g. author, mentioned, project roles).

    extraction_fields() ¶

    Required vs optional metadata fields the analysis pass should fill.

    naming() ¶

    The naming convention id (e.g. snake_case_iso_dates).

    synthesis() ¶

    The project-synthesis template: name, title, ordered sections, prose rules.

    load_profile(name, profiles_dir) ¶

    Load and validate {profiles_dir}/{name}.toml.

    Raises FileNotFoundError if the profile file does not exist, and ValueError if the TOML is malformed or fails schema validation.
